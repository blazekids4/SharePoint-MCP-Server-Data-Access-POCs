# Meeting Notes — March 24, 2026

### Meeting Summary

- The team discussed progress on integrating an MCP server with Azure AI Search to enhance code and document search capabilities across SharePoint and GitHub, focusing on building an MVP that supports intelligent, tokenized, and cached search results for internal users.
- Saurabh demonstrated a prototype that uses Azure AI Search for indexing and searching code snippets, supporting advanced queries, result scoring, and caching for performance. The demo included mocked SharePoint data due to lack of direct access.
- The group emphasized the importance of access control, ensuring users only see results for repositories and documents they are permitted to access, with plans to inherit permissions from SharePoint, GitHub, and Entra ID.
- There was consensus to start with a single developer group and a consolidated SharePoint site for the MVP, then expand to other groups and sources as the process matures.
- The team discussed the need to compare Azure AI Search, Microsoft Graph, and Copilot Retrieval API approaches for indexing and retrieval, considering factors like latency, complexity, and cost.
- The current focus is on understanding available tools and defining user stories and requirements before finalizing the MVP scope.

### Key Decisions

- The MVP will initially use Azure AI Search for code and document indexing, with a focus on internal SharePoint and GitHub sources.
- Access control will be enforced based on existing permissions in SharePoint, GitHub, and Entra ID, rather than building a new system.
- The team will start with one developer group and a primary SharePoint site, expanding to others after validating the approach.
- Multiple indexing and retrieval solutions (Azure AI Search, Microsoft Graph, Copilot Retrieval API) will be evaluated in parallel.
